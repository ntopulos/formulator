Introduction

This section is intended to shed light on the structure of Formulator. If you do not plan to extend it, or use it in any advanced way, you probably do not need to read the following.

Classes description

Formulator

Main class, an object represents a given form on the page.

Elements

Classes describing elements, each one extends - directly or indirectly - the main Element class. (called by Formulator)

Render

Helper with static methods generating the HTML elements. (called by Formulator and Elements)

Rules

Class with the rules as static methods. (called by Formulator)

Objects and data struture

A Formulator object contains Elements objects, each one of them corresponds to a given element (HTML) of the form.

But please note that in the case of multirow forms, each Element object describes all the same form elements from the "column". The following structure will be the same for 1, 2 or 100 rows:


Formulator Object yourfrom
    Element Object firstname
    Element Object lastname

The only differences between the rows are limited to the input and the validations results. This data is stored within each Element object in an array containing the data for the whole "column", row by row:


[data] => Array
    (
        [0] => Array
            (
                [content] => John
                [valid] => 1
                [error] => 
            )
        [1] => Array
            (
                [content] => 
                [valid] => 
                [error] => Required.
            )
    )
