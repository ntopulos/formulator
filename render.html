Introduction

This section covers the rendering part of a Formulator instance.

Formulator renders separately all the elements that make the form, thus you have two options: use the separated elements with your own template system (Rendering) or let Formulator render a complete form (pseudoTemplate rendering).

While the second option can be useful in some cases, usually the first one will be preferred. It allows your software to stick to its conventions, and in addition it makes use of your framework's caching capabilities.

Rendering

After the computation of your form, you can access the render result:


if ($formulator->validate()) { }
$render = $formulator->render();

This method outputs an array with the following content:

$render['open']
Opening tag and general elements of the form.
$render['elements']
All Elements objects are given there.
$render['buttons']
Buttons to be rendered at the end of the form.

The next step is to provide this data to a template. To create your own template, the easiest way is to have a look at the two provided with Formulator.

Please note that using $render['open'] is NOT optional: this variable contains mandatory information for the proper functioning of Formualtor.

Template rendering

Formulator has an integrated template feature:


if ($formulator->validate()) { }
$render_str = $formulator->renderTemplate();

In the above case $render_str is no longer an array, but a string that contains the whole HTML form. You can directly echo it.

Styles

Example of CSS styles designed for the provided views:


/* Formulator CSS */
 
form {
    display: table;
}
 
div.invalid input,
div.invalid textarea {
    border-color: #ED591A;
}
form .error {
    color: #ED591A;
}
 
form > div {
    display: table-row;
}
label,
form div div {
    display: table-cell;
}
label {
    padding: 0.5em 3em 0.5em 0.5em;
}
form > div:last-of-type div {
    padding-top: 1em;
}
