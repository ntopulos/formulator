Introduction

An element is an object that represents an HTML form element and all the related informations: validation rules, validation results, values, etc...

All elements are stored in the elements object, but note that you always have to access them by their table. The manually added elements are referenced to root table, while this value corresponds to the database table for the automatically generated elements.


// manually adding an element
$formulator->addElement('username');

// accessing the element to add a rule
$formulator->elements->root->username->addRule('alpha_numeric');

Methods

addElement($name[, $type, $default_value, $table, $options])
Adds an element.
assisted elements generation methods

Parameters

$name
Name of the element, must be a valid PHP variable/method name.
$type
Formulator element type, see the elements reference.
$default_value
The default value, can be null.
$table
The table of the element, by default it is root.
$options
Options of the elements as described here, but starting in the table dimension since this information is already specified by method itself.
These options can also be added later with specific methods.

Attributes

Attributes of the HTML form elements are automatically inherited from validation rules. But it is also possible to add them manually. If a conflict occurs, the manually added attribute will overwrite the inherited one.

Methods

addAttribute($attribute)
Adds an attribute to the element.
If a conflict occurs the new attribute replaces the existing one.
setAttribute($attribute)
Performs the same task as addAttribute(), but deletes all existing elements attributes first.

Parameters syntax

Arguments formats for both methods:


// -- String format --
// attribute without argument
$attribute = array('readonly');
// attribute with argument
$attribute = array('maxlength="3"');

// -- Array format --
// attribute without argument
$attribute = array('readonly' => null);
// attribute with argument
$attribute = array('maxlength' => 3);

Chaining

The two previous methods can be chained like the rules methods and with them.
